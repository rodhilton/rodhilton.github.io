You've got code, you've got a suite of tests. You want to do some refactoring, but you arent usre if there are tests covering what you're about to do.

Simple solution.

take what you want to refactor, and add an additional step before refactoring it. comment out the line and instead make it return an incorrect value. intentionally break the code, then run the tests. if they all pass, you're not safe, write some new tests. if a test breaks, your refactoring is safe to do (if it breaks something, you'll know)
